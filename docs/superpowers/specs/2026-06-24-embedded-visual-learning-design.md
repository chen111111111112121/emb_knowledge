# 「看得懂的嵌入式」交互动画学习网站 — 设计文档

> 日期:2026-06-24
> 状态:已与用户确认,待转入实现计划

## 1. 背景与目标

用户具备**嵌入式软件背景**(前后端不熟,但有 Claude Code 辅助),启动资金几乎为零,面向**国内市场**,目标是**副业 / 被动收入**。

经过调研,确定走「**主线一:嵌入式知识/内容付费** + **顺手二:用 AI 做垂直网站/工具**」的组合。本设计文档聚焦其中第一个可落地、可用 Claude Code 构建的产品。

### 产品一句话定位
一个用**交互动画**把抽象的嵌入式协议/算法讲给新手听的学习网站——「看得懂的嵌入式」(暂定名)。

### 为什么是这个产品
- **差异化**:国内嵌入式可视化讲解的优质站点很少(蓝海);在线计算器类工具是红海。
- **发挥专长**:用户既懂原理,又懂新手在哪卡壳,这是护城河。
- **打通两条变现线**:免费动画负责引流 + SEO + 传播(动画极易被分享);付费部分是配套的系统课程 / 源码 / 知识星球。
- **命中高付费人群**:入门 / 转行的学生、应届生,最需要"看得懂"。

## 2. 目标用户

嵌入式**入门 / 转行**的学生和应届生。特征:看文档/时序图看不懂,需要直观的动态演示才能"秒懂"。

## 3. 范围(MVP)

> 原则:第一版**只做 1 个概念,但做到极致**。一个让新手"原来是这样"的爆款动画,胜过十个平庸的工具。

### 首发功能:UART 串口通信可视化(MVP 唯一功能)

一个单页交互动画,演示"一个字节如何一位位变成电平信号发出去"。

**用户输入 / 配置**
- 输入:一段字符 或 一个字节(支持十六进制/字符输入)
- 配置项:
  - 波特率(如 9600 / 115200 等常见值,可选)
  - 数据位:5–9 位
  - 校验位:无 / 奇校验 / 偶校验
  - 停止位:1 / 2 位

**核心动画**
- 点击"发送"后,在时间轴上绘制 **TX 线电平波形**,逐位高亮播放:
  `空闲(高电平) → 起始位(低) → 数据位(LSB 在前,逐位点亮) → 校验位 → 停止位(高)`

**同步信息展示**
- 当前字节的二进制表示
- 文字解说:当前正在发送哪一位、代表什么
- 每一位的时长(= 1 / 波特率)

**播放控制**
- 速度可调:慢放 / 单步,让新手看清每一位

**辅助**
- 旁边配一段简明的 UART 原理讲解文字
- 移动端适配

### MVP 明确不做(YAGNI)
- 不做用户登录 / 账号系统
- 不做后端 / 数据库
- 不接广告(网站广告变现弱,留到小程序阶段)
- 不做其它协议/算法(I2C、CRC、PID 等留到后续迭代)
- 不做支付 / 付费墙(MVP 阶段只验证流量与口碑)

## 4. 技术方案

对新手 + Claude Code 友好,且为"后续搬微信小程序"铺路。

- **纯静态网站**:无后端、无数据库 → 零成本、零运维。
- **技术栈**:HTML + CSS + 原生 JavaScript;用 Canvas 或 SVG 绘制波形动画(实现时择一,优先实现简单度)。
- **核心逻辑与渲染分离**:
  - `uart.js`(或同类模块):**纯函数**,输入"字节 + 配置",输出"位序列(带类型标注:起始/数据/校验/停止)"。不依赖 DOM。
  - 渲染层:负责把位序列画成波形 + 动画播放控制。
  - **理由**:纯逻辑模块以后搬微信小程序可直接复用,只需重做界面。
- **页面结构预留扩展**:
  - 首页:概念列表(MVP 只有 UART 一项)
  - 子页 `/uart`:UART 可视化
  - 以后加新概念 = 加一个子页(`/i2c`、`/crc` ……)
- **部署**:Cloudflare Pages / Vercel / GitHub Pages 等免费静态托管(海外托管不用备案)。

### 组件边界(各自单一职责)
| 单元 | 职责 | 依赖 |
|------|------|------|
| `uart.js` 逻辑模块 | 字节+配置 → 带类型标注的位序列(纯函数) | 无 |
| 波形渲染器 | 把位序列画成电平波形 | Canvas/SVG |
| 播放控制器 | 控制逐位高亮、速度、单步、播放/暂停 | 渲染器 |
| 配置/输入 UI | 收集用户输入与配置 | 逻辑模块 |
| 页面框架 | 首页 + 子页路由、讲解文案、移动端布局 | 上述各单元 |

## 5. 变现路径

- **MVP 阶段**:不接广告、不收费。专注把 UART 动画做到直观漂亮,攒口碑和流量。
- 页面预留"**想系统学 → 公众号 / 知识星球**"入口,为方向一(内容付费)引流。
- **后续**:积累出多个动画 + 稳定流量后,把热门概念搬**微信小程序挂流量主**变现(`uart.js` 等纯逻辑可复用)。
- 长期飞轮:免费动画(引流) → 个人品牌 → 付费内容(星球/课程/源码)+ 小程序广告。

## 6. 上线后第一步引流

做好后发布到 **B站(录屏讲解)+ 公众号 + 掘金/CSDN**,标题主打"一个动画看懂 UART 串口通信"。根据数据反馈决定下一个做哪个概念。

## 7. 成功标准

- MVP:UART 动画能让一个没学过串口的新手看完后说得清"起始位/数据位/校验位/停止位"分别是什么。
- 动画直观、流畅、移动端可用。
- 至少在一个平台(B站/公众号/掘金)发布并获得真实反馈。
- 核心逻辑 `uart.js` 与渲染层解耦,可被未来的小程序版本复用。

## 8. 后续迭代(MVP 之后,非本次范围)

- 增加更多概念动画:I2C/SPI 时序、CRC 校验过程、PID 控制等
- 热门概念搬微信小程序,接入流量主广告
- 沉淀付费内容(系统课程 / 配套源码 / 知识星球)
