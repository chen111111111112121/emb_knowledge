# UART 教学内容深化 + 多芯片配置代码 — 设计文档

> 日期:2026-06-24
> 状态:已与用户确认,待转入实现计划
> 关联:在 [2026-06-24-embedded-visual-learning-design.md](2026-06-24-embedded-visual-learning-design.md) 的 UART MVP 基础上扩展

## 1. 背景与目标

UART MVP(交互动画)已上线,但教学内容偏薄。用户决定**把重点放在做好教学内容**,要求:

1. 把 UART 的教学内容做**系统、深入**;
2. 增加**主流芯片的 UART 配置方法 + 可运行代码**参考。

目标人群仍是嵌入式入门/转行的学生——既要"看懂原理",也要"知道在真实芯片上怎么配、怎么写"。

## 2. 范围

### 覆盖芯片(第一版 4 个)
- **STM32**(HAL 库,开发环境 STM32CubeIDE)
- **ESP32**(Arduino 框架,开发环境 Arduino IDE)
- **51 单片机**(STC,寄存器级,开发环境 Keil C51)
- **Nordic nRF52**(开发环境与驱动以官方文档为准,实现时用 context7 核对)

### 教学深度:系统深入
### 代码形式:可运行最小示例 + 逐行注释

## 3. 页面结构(`uart.html` 升级为完整教学页)

```
顶部:标题 + 交互动画(保留现有,不改动画逻辑)
├─ 1. UART 是什么          异步串行通信;与 SPI/I2C 一句话对比
├─ 2. 一帧长什么样         配合动画:空闲/起始/数据(LSB先)/校验/停止;8N1 记法
├─ 3. 波特率与误差         采样原理(过采样/中点采样)、为什么必须匹配、误差来源、容限举例
├─ 4. 电平与接口           TTL / RS-232 / RS-485 区别、必须共地、TX-RX 交叉接线
├─ 5. 校验与可靠性         奇偶校验能查 1 位错不能纠错;帧错误 / 溢出错误
├─ 6. 流控                 无流控 / 软件流控(XON/XOFF)/ 硬件流控(RTS/CTS)
├─ 7. 常见 Bug 排查表       乱码、只收不发、完全无数据、偶尔丢字节、中文乱码……
└─ 8. 主流芯片怎么配        Tab:STM32 / ESP32 / 51 / nRF52
        每芯片:外设&引脚 → 配置要点 → 可运行最小示例(逐行注释)→ 常见坑
```

各章节内容要求(系统深入):
- 第 3 章要解释"双方约定相同波特率"的原理、UART 接收端如何用过采样在每位中点采样、波特率误差来自时钟分频不能整除,并给一个误差计算示例。
- 第 4 章要讲清 MCU 的 TTL 电平(3.3V/5V)、RS-232 的 ±12V 反相、RS-485 的差分长距离多机,以及"两设备通信必须共地""TX 接对方 RX"。
- 第 7 章用**表格**呈现:症状 → 可能原因 → 排查方法,覆盖新手最常踩的坑。

## 4. 代码组织

采用"图文写进 HTML + 芯片代码用结构化数据 + 本地语法高亮"方案:

| 文件 | 职责 | 依赖 |
|------|------|------|
| `uart.html` | 教学正文(图文穿插,语义化 HTML,利于 SEO)+ 芯片区容器 | styles |
| `src/chips/uart-chips.js` | 结构化数据:每芯片一个对象(纯数据,无 DOM) | 无 |
| `src/chip-tabs.js` | 读数据渲染 Tab 与代码块、绑定切换、触发高亮 | uart-chips.js, highlight.js, DOM |
| `vendor/highlight.min.js` | 本地语法高亮库(支持 C/C++/Arduino),避免 CDN 被墙 | 无 |
| `vendor/highlight.css` | 高亮主题样式 | 无 |
| `styles/main.css` | 新增教学排版、Bug 表格、Tab、代码块样式 | 无 |

`uart-chips.js` 中每个芯片对象的结构:
```js
{
  id: 'stm32',
  name: 'STM32 (HAL)',
  mcu: 'STM32F1/F4 系列',
  env: 'STM32CubeIDE',
  peripheral: 'USART1',
  pins: 'PA9 (TX) / PA10 (RX)',
  configNotes: '配置要点文字……',
  code: '可运行最小示例(字符串,含逐行注释)',
  language: 'c',            // 给高亮用
  pitfalls: ['常见坑1', '常见坑2'],
}
```

`chip-tabs.js` 的纯逻辑(数据 → DOM 片段的构建函数,如 `buildTabButton`/`buildPanel` 返回字符串或节点)与"挂载到页面/绑定事件"分离,前者可单测。

## 5. 代码准确性保证

4 个芯片的示例**逐个用 context7 拉官方文档/库 API 核对**后再写入,不凭记忆:
- STM32:`HAL_UART_Init` / `HAL_UART_Transmit` / `HAL_UART_Receive` 签名与用法
- ESP32:Arduino `Serial.begin(baud, config, rx, tx)` / `Serial.read` / `Serial.print`
- 51:`SCON` / `TMOD` / `TH1` 用定时器1 产生波特率,`SBUF` / `TI` / `RI`
- nRF52:官方 UART 驱动 API 与初始化流程

每段代码顶部注释标注:开发环境、适用芯片、引脚。

每个芯片的"最小示例"统一覆盖:**初始化 → 发送一串字符 → 接收一个字节并回显**。

- Bug 排查表覆盖新手最常见的至少 5 类问题。
